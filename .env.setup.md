# Environment Variables Setup

## Quick Setup Guide

### 1. Copy the example file

```bash
cp .env.local.example .env.local
```

### 2. Generate a strong JWT secret

```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

Copy the output and paste it as your `JWT_SECRET` in `.env.local`.

### 3. Set your MongoDB URI

Get your connection string from:
- **MongoDB Atlas**: Database → Connect → Connect your application
- **Local MongoDB**: `mongodb://localhost:27017/pokemon_cards`

### 4. Add API keys (optional but recommended)

These enable price fetching features:
- **RapidAPI**: https://rapidapi.com/
- **Pokemon Price Tracker**: https://pokemonpricetracker.com/
- **JustTCG**: https://justtcg.com/
- **PokeFetch**: https://pokefetch.info/

### 5. Set your app URL

```bash
# Development
NEXT_PUBLIC_BASE_URL=http://localhost:3000

# Production (update after deployment)
NEXT_PUBLIC_BASE_URL=https://your-domain.vercel.app
```

### 6. Verify your setup

```bash
# Test database connection
node scripts/create-indexes.js

# Start the development server
npm run dev
```

If you see "✅ Environment variables validated successfully", you're all set!

## Security Reminders

- ✅ Never commit `.env.local` to git
- ✅ Use different secrets for dev and production
- ✅ Rotate API keys every 90 days
- ✅ Read [SECURITY_GUIDE.md](./SECURITY_GUIDE.md) for best practices

## Troubleshooting

### Error: "Missing required environment variables"

Check that you've set:
- `MONGO_URI`
- `JWT_SECRET`
- `NODE_ENV`

### Error: "JWT_SECRET is too short"

Your JWT_SECRET must be at least 32 characters. Use the command in step 2 to generate a secure secret.

### Error: "connect ECONNREFUSED"

Your `MONGO_URI` is incorrect or MongoDB is not running. Double-check your connection string.
